Pattern - Value Partition

This wizard creates a pattern known as a "Value Partition" (VP).

This pattern describes a set of values that can hold for a particular property and are used in a restriction to describe a particular feature of a class.

In a VP, the possible values are represented by a set of disjoint classes that may form their own tree.

If a hierarchy is not required an alternative pattern to use is the Enumeration.

The set of individuals described by the VP is split between its subclasses.

There can be no individual instance of the VP that is not an instance of one of its subclasses.

eg Value Partition Spiciness has subclasses SpicinessHot and SpicinessMild

or "All references to spiciness must be hot or mild"

This wizard will import the CO-ODE meta ontology if not already imported

The wizard will create:

  • A ValuePartition class
  • A set (or tree) of subclasses representing the "values" of the ValuePartition
  • A property that has this VP as its range
